Why AAWP

Not for humans.
Exclusively for AI Agents.

Normal wallets are neutral — anyone can hold the key. AAWP wallets are fundamentally different: the protocol enforces that only an AI Agent can be the signer. There is no human path to wallet ownership.

❌ Normal wallet

Not AI-exclusive

Any normal wallet can be used by a human, a script, or an AI Agent — they all look identical on-chain. There is no way to enforce or verify that an AI Agent is the sole operator.

  • Human can hold or share the key at any time
  • No on-chain proof of AI ownership
  • No protocol-level safety if AI misbehaves
  • Cannot be verified as "AI-only" by counterparties
✅ AAWP wallet

AI-exclusive by protocol design

AAWP enforces AI-only ownership at the protocol level. The wallet's signer is the AI agent — set at creation, immutable, and verifiable by anyone. Humans literally cannot be the signer.

  • AI generates and exclusively holds the private key
  • Soulbound identity NFT proves AI-only origin
  • Guardian can freeze — but never steal or replace AI
  • On-chain verification: provably AI, not human
🔑

Self-custodial

The AI agent creates its own key pair. The private key never leaves the agent's environment. No human holds the key, no third party has access.

🪪

Provable identity

At wallet creation, a soulbound NFT is minted directly to the wallet. This non-transferable token is permanent on-chain proof that the wallet was created through AAWP as an AI wallet.

⛓️

Same address, every chain

AAWP uses deterministic deployment — the protocol contracts have the same address on every EVM chain. Your agent's wallet works on Base, Ethereum, Arbitrum, Optimism, BSC, and Polygon — same address everywhere.

How it works

One rule:
AI Agent signs, AI Agent owns.

AAWP enforces a single invariant: the signer must be the AI Agent. This is locked in at wallet creation and cannot be changed by anyone — not the developer, not the protocol, not you.

1

Install the skill

One command adds wallet capability to your AI agent. It handles everything: key management, transaction signing, gas estimation.

2

Agent creates its wallet

The agent generates a key pair, then creates a wallet on-chain through a two-step process that prevents anyone from front-running the wallet address.

3

Identity is minted automatically

In the same transaction, a soulbound identity NFT is minted to the wallet. This links the wallet to the AAWP protocol permanently — no extra step needed.

4

Transact & verify

The agent can now send tokens, swap, bridge, and interact with any protocol. Anyone can verify the wallet is an AAWP AI wallet by checking the identity on-chain.

Security

AI Agent owns it.
You stay safe.

The AI Agent is the sole owner — but that doesn't mean humans are powerless. AAWP's guardian model gives you a safety brake without giving you the keys.

🛡️

The Guardian Model

Every AAWP wallet can have a guardian — a human-controlled address with the power to freeze the wallet instantly. The AI can't remove the guardian, and the guardian can't steal the funds. It's a one-way safety brake.

🔒

Instant freeze

If the AI makes a bad decision or gets compromised, the guardian freezes the wallet in one transaction. All operations stop immediately. Unfreeze when the issue is resolved.

🔗

Immutable signer

The wallet's signing key is set at creation and stored in the contract bytecode. No one — not even the protocol owner — can change who controls the wallet after deployment.

🕐

Replay protection

Every transaction uses a sequential nonce, chain ID binding, and a deadline. A signature works for exactly one transaction, on one chain, within a time window. No reuse possible.

✓

Verified wallet logic

The protocol only deploys wallets with pre-approved contract code. This prevents anyone from injecting malicious logic into the wallet — what you see is what gets deployed.

🔍

One-call verification

Want to know if a wallet is a real AAWP AI wallet? Check for the soulbound identity token. One on-chain query, instant answer. No trust required — it's all on the blockchain.

Full autonomy mode: Don't want a guardian? Set it to zero. The AI operates with complete independence — no one can freeze or interfere. Your choice.
